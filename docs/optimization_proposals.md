#### 2.2 企业选择器优化

**当前问题：**
```ruby
# 当前实现：N+1查询问题
@companies_with_announcement_count = @companies.map do |company|
  # 每个企业都执行一次查询
  announcement_count = AnnouncementService.new(
    user: current_lawyer,
    company_ids: [company.id]
  ).call[:combined_announcements].count
  
  # 返回企业对象+公告数量
  OpenStruct.new(
    id: company.id,
    name: company.name,
    announcement_count: announcement_count
  )
end

问题：50个企业 = 50次数据库查询 = 慢！
```

**优化后：**
```ruby
# 使用数据库聚合查询
announcement_counts = Announcement
  .where(dismissed: false)
  .group(:company_id)
  .count

@companies = Company.ordered.map do |company|
  company.announcement_count = announcement_counts[company.id] || 0
  company
end

结果：50个企业 = 2次数据库查询 = 快！
```

**具体改动：**
- 文件：`app/controllers/lawyer/companies_controller.rb`
- 改动：优化数据查询逻辑
- 代码量：约20行

**预期效果：**
- 页面加载速度提升70%（从2秒降到0.6秒）
